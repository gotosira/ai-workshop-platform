# Lesson 1.4: Iterative Refinement Techniques

**ระยะเวลา:** 40 นาที  
**Level:** 🌱 Beginner

---

## 🎬 HOOK (30 วินาที)

> "Designer ไม่เคยได้งานที่ Perfect ตั้งแต่ Draft แรก
> เราทำ Iteration — ปรับ รับ Feedback แก้ไข วนซ้ำ
> 
> การใช้ AI ก็เหมือนกัน — Prompt แรกแทบไม่เคยสมบูรณ์
> แต่รู้วิธี Iterate แล้ว คุณจะได้ผลลัพธ์ที่ต้องการทุกครั้ง"

---

## 🎯 LEARNING OBJECTIVES

หลังจบ Lesson นี้ คุณจะสามารถ:
1. ใช้เทคนิค Iterative Prompting
2. ให้ Feedback กับ AI อย่างมีประสิทธิภาพ
3. รู้ว่าเมื่อไหร่ควร Restart vs. Refine

---

## 📚 CONTENT

### Part 1: Mindset — AI เป็น Conversation ไม่ใช่ One-shot (5 นาที)

**❌ Mindset เก่า:**
"ต้องเขียน Prompt ที่ Perfect ตั้งแต่แรก"

**✅ Mindset ใหม่:**
"Prompt แรก = Starting point → Iterate จนได้ที่ต้องการ"

**เปรียบเทียบ:**
| One-shot Mindset | Iterative Mindset |
|------------------|-------------------|
| กดส่ง รอลุ้น | กดส่ง แล้ว Refine |
| ผิดหวังถ้าไม่ตรง | คาดหวังว่าต้องปรับ |
| เสียเวลาคิด Prompt นาน | เริ่มเร็ว ปรับเร็ว |

---

### Part 2: Iteration Techniques (20 นาที)

#### 🔹 Technique 1: "ดีแล้ว แต่..."

```
[หลังจากได้ Output แรก]

"ดีมาก แต่ช่วยปรับ:
- Tone ให้ Casual มากขึ้น
- ตัดส่วนที่ยาวเกินไป
- เพิ่ม Emoji 1-2 ตัว"
```

**เมื่อไหร่ใช้:** Output ใกล้เคียง แต่ต้องปรับ Details

---

#### 🔹 Technique 2: "เลือก A, ปรับแบบ B"

```
"ชอบ Option 2 มากที่สุด
ช่วยขยาย Option 2 โดย:
- เพิ่มรายละเอียด
- ใช้ Tone แบบ Option 3"
```

**เมื่อไหร่ใช้:** มีหลาย Options และต้องการ Combine

---

#### 🔹 Technique 3: "ให้อีก 5 แบบ แต่..."

```
"ให้อีก 5 ตัวเลือก แต่:
- เน้น Playful มากขึ้น
- สั้นกว่าเดิม
- หลีกเลี่ยงคำว่า 'innovative'"
```

**เมื่อไหร่ใช้:** ต้องการ Explore ทิศทางใหม่

---

#### 🔹 Technique 4: Specific Feedback

**❌ Feedback ไม่ดี:**
```
"ไม่ชอบ ทำใหม่"
```

**✅ Feedback ดี:**
```
"ปัญหา: 
- Headline ยาวเกินไป (ไม่เกิน 6 คำ)
- Tone ดูเป็นทางการเกินไป สำหรับ Gen Z

ช่วยแก้ 2 จุดนี้ โดยคง Structure เดิม"
```

---

#### 🔹 Technique 5: Role Shift

```
[Output ที่ได้]

"ตอนนี้สมมติว่าคุณเป็น UX Critic 
Review output ที่เพิ่งให้ไป
- อะไรที่ดี?
- อะไรที่ยังพัฒนาได้?
- ให้ version ที่ปรับปรุงแล้ว"
```

**เมื่อไหร่ใช้:** ต้องการ Self-critique และ Improvement

---

#### 🔹 Technique 6: Constraint Addition

```
"ดี แต่ลองทำใหม่โดย:
- ห้ามใช้คำว่า 'seamless'
- ต้องมี Call-to-action
- ความยาวลดลง 50%"
```

**เมื่อไหร่ใช้:** Output ดี แต่ต้องการ Tighten

---

### Part 3: เมื่อไหร่ควร Restart vs. Refine (10 นาที)

#### ✅ **Refine (ต่อยอด)** เมื่อ:
- Output ถูกทิศทาง แค่ต้องปรับ Details
- มี Parts ที่ดี อยากเก็บไว้
- ต้องการ Consistency กับที่ได้มาแล้ว

**วิธี:** ใช้ Techniques ข้างบน

---

#### 🔄 **Restart (เริ่มใหม่)** เมื่อ:
- Output ผิดทิศทางโดยสิ้นเชิง
- Context ผิด ทำให้ทุกอย่างผิด
- Iterate ไป 3-4 รอบแล้วยังไม่ถูก

**วิธี:** 
```
"เริ่มใหม่ ลืมทุกอย่างที่คุยกันมา

[Prompt ใหม่ที่ชัดเจนกว่าเดิม]"
```

---

### Part 4: Iteration Workflow (5 นาที)

```
┌─────────────────────────────────────┐
│  1. First Prompt (80% complete)     │
└─────────────┬───────────────────────┘
              ↓
┌─────────────────────────────────────┐
│  2. Review Output                   │
│     - What's good?                  │
│     - What needs change?            │
└─────────────┬───────────────────────┘
              ↓
┌─────────────────────────────────────┐
│  3. Specific Feedback               │
│     "ดี แต่ปรับ X, Y, Z"            │
└─────────────┬───────────────────────┘
              ↓
┌─────────────────────────────────────┐
│  4. Repeat 2-3 times max            │
│     (ถ้าเกินนี้ = Restart)           │
└─────────────┬───────────────────────┘
              ↓
┌─────────────────────────────────────┐
│  5. Final Output ✓                  │
└─────────────────────────────────────┘
```

## 🎮 DEMO

**Live Demo: Full Iteration Cycle**

**Prompt แรก:**
```
เขียน Headline สำหรับ Landing Page ของ AI Writing Tool
```

**Output 1:** "Revolutionize Your Writing with AI"

**Iteration 1:**
```
ดี แต่:
- ภาษาอังกฤษค่อนข้าง Generic
- ลองเป็นภาษาไทยที่ฟังดู Modern
- เน้น Benefit: ประหยัดเวลา
```

**Output 2:** "เขียนได้เร็วกว่าเดิม 10 เท่า ด้วย AI"

**Iteration 2:**
```
ใกล้เคียงแล้ว แต่:
- "10 เท่า" อาจ Over-promise
- อยากได้ Tone ที่ Approachable มากกว่า
- ลองเพิ่ม Curiosity element
```

**Output 3:** "งานเขียนที่เคยใช้ชั่วโมง ตอนนี้เสร็จในนาที"

**Final:** ✅ ได้แล้ว!

## ✏️ PRACTICE

**แบบฝึกหัด: Iteration Chain**

**Prompt เริ่มต้น:**
```
เขียน CTA Button Text สำหรับ Free Trial
```

**Output 1:** "Start Free Trial"

**คำถาม:** 
1. อะไรที่ดี?
2. อะไรที่ปรับได้?
3. เขียน Iteration feedback

**ตัวอย่าง Iteration:**
```
ดี แต่ปรับ:
- เป็นภาษาไทย
- เพิ่ม Urgency เล็กน้อย
- บอก Duration ของ Trial (14 วัน)
```

## 📌 SUMMARY

**Key Takeaways:**

1. **Iteration = Normal** — อย่าคาดหวัง Perfect ตั้งแต่แรก
2. **Specific Feedback** — บอกชัดว่าอะไรดี อะไรต้องแก้
3. **2-3 Iterations Max** — ถ้าเกินนี้ ควร Restart
4. **ผสม Techniques** — เลือก A ปรับแบบ B เพิ่ม C
