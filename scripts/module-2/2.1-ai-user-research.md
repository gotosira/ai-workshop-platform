### Part 2: AI ช่วย Research Planning (10 นาที)

#### 🔹 สร้าง Interview Questions

```
# Context
กำลังทำ Research สำหรับ:
- Product: Food Delivery App
- Goal: เข้าใจ Pain Points ในการสั่งอาหาร
- Users: คนทำงาน 25-40 ปี ในกรุงเทพ

# Task
สร้าง Interview Guide ที่มี:
- Warm-up questions (2-3)
- Main questions เกี่ยวกับ Behavior (5-7)
- Questions เกี่ยวกับ Pain Points (4-5)
- Closing questions (2)

# Format
แต่ละ Question ให้มี:
- Question
- Purpose (ทำไมถามข้อนี้)
- Follow-up prompts
```

---

#### 🔹 สร้าง Screener Survey

```
# Context
ต้องการหา Participants สำหรับ Research:
- Product: Investment App สำหรับมือใหม่
- Target: คนอายุ 22-35 ที่เพิ่งเริ่มลงทุน

# Task
สร้าง Screener Survey ที่ Filter ได้:
- มีประสบการณ์ลงทุน < 2 ปี
- มี Income ระดับ Middle
- สนใจเรียนรู้เรื่องการลงทุน
- ไม่ทำงานในสาย Finance

# Format
- 8-10 questions
- Multiple choice ส่วนใหญ่
- บอก Qualifying criteria ของแต่ละข้อ
```

### Part 3: AI ช่วย Synthesis & Analysis (15 นาที)

#### 🔹 วิเคราะห์ Interview Transcripts

```
# Context
นี่คือ Transcript จาก User Interview 5 คน
เกี่ยวกับประสบการณ์ใช้ E-commerce App

# Task
วิเคราะห์และหา:
1. Common Pain Points (ปัญหาที่หลายคนพูดถึง)
2. Key Behaviors (พฤติกรรมสำคัญ)
3. Unmet Needs (ความต้องการที่ยังไม่ถูกตอบ)
4. Quotes ที่น่าสนใจ

# Format
- จัดกลุ่ม Theme
- บอกจำนวนคนที่ mention แต่ละ Theme
- แนบ Direct quotes

[Paste Transcripts]
```

---

#### 🔹 สร้าง Affinity Diagram

```
# Context
นี่คือ Notes จาก User Research:
[Paste all notes/observations]

# Task
จัดกลุ่มเป็น Affinity Diagram:
1. Group similar observations
2. Name each group
3. Identify hierarchy (main themes → sub-themes)

# Format
Main Theme 1: [Name]
├── Sub-theme A
│   - Observation 1
│   - Observation 2
├── Sub-theme B
│   - Observation 3
...
```

---

#### 🔹 สร้าง Persona จาก Data

```
# Context
จาก Research กับ Users 10 คน พบ Pattern ดังนี้:
[Summary of patterns]

# Task
สร้าง 2-3 Personas ที่แตกต่างกัน

# Format แต่ละ Persona:
- Name & Photo description
- Demographics
- Goals (3)
- Frustrations (3)
- Behaviors (3-5)
- Quote ที่สะท้อน Mindset
- Scenario ตัวอย่าง
```

### Part 4: ข้อจำกัดที่ต้องรู้ ⚠️ (5 นาที)

**❌ สิ่งที่ AI ทำไม่ได้:**

1. **แทนที่การคุยกับ User จริง**
   - AI ไม่รู้ Context เฉพาะของ Users คุณ
   - Non-verbal cues หายไป

2. **สร้าง User Insights จากศูนย์**
   - AI Hallucinate ได้
   - ต้องมี Real data ก่อน

3. **ตัดสินใจแทนคุณ**
   - AI ให้ Analysis
   - Human ตัดสินใจว่า Insight ไหนสำคัญ

**✅ Best Practice:**
- ใช้ AI เป็น "Research Assistant" ไม่ใช่ "Researcher"
- Validate AI insights กับ Real data
- Human judgment เสมอสำหรับ Final decisions

## 📌 SUMMARY

**Key Takeaways:**

1. **AI = Research Assistant** — ช่วย ไม่ใช่แทนที่
2. **ช่วยได้มากตอน Synthesis** — Pattern finding, Grouping
3. **ต้องมี Real Data** — AI วิเคราะห์จาก Data ที่คุณให้
4. **Human Judgment สำคัญ** — ตัดสินใจ Insights สำคัญ
