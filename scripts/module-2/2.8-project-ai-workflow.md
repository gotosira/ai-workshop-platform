## 🎯 PROJECT OBJECTIVES

เมื่อจบ Project นี้ คุณจะมี:

1. ✅ **Working AI Workflow** — ที่ทดสอบแล้วว่าใช้ได้จริง
2. ✅ **Prompt Library** — 3-5 prompts ที่ใช้ซ้ำได้

### Prompt Template Format

**สำหรับแต่ละ AI Step สร้าง Template:**

```markdown
## Prompt Template: [Name]

### 📌 When to Use
[อธิบายว่าใช้ตอนไหน ในสถานการณ์อะไร]

### 📥 Required Inputs
- [ ] [Input 1]: [description]
- [ ] [Input 2]: [description]
- [ ] [Input 3]: [description]

### 📝 The Prompt

```
# Context
[Background information about the task]
[Project/brand context]

# Input
[Placeholder for user input]

# Task
[Clear instructions on what AI should do]

# Output Format
[Specify how output should be structured]

# Constraints
[Any limitations or requirements]
```

### ✅ Expected Output
[อธิบายว่าควรจะได้ output หน้าตาประมาณไหน]

### 🔄 Iteration Tips
- ถ้าผลลัพธ์ยาวเกินไป → [tip]
- ถ้าผลลัพธ์ไม่ตรง tone → [tip]
- ถ้าขาดรายละเอียด → [tip]

### 💡 Pro Tips
- [Tip 1]
- [Tip 2]
```

---

### Example: Research Synthesis Prompt

```markdown
## Prompt Template: Interview Synthesis

### 📌 When to Use
หลังจากทำ User Interview เสร็จ 3-5 คน ต้องการสรุป Key findings

### 📥 Required Inputs
- [ ] Interview transcripts หรือ notes (3-5 interviews)
- [ ] Research objectives
- [ ] Target user description

### 📝 The Prompt

```
# Context
ฉันเป็น UX Researcher ทำ User interviews สำหรับ [product]
Target users: [description]
Research objectives: [list]

# Input
Interview notes จาก [X] participants:

[Paste notes here]

# Task
วิเคราะห์ Interview notes และสรุป:
1. Key themes (grouped by frequency)
2. Notable quotes (5-7 most insightful)
3. Pain points (prioritized by severity)
4. Unmet needs
5. Opportunities for design

# Output Format
## Key Themes
### Theme 1: [Name]
- Frequency: X/Y participants
- Key insight: [1 sentence]
- Supporting quotes: [2 quotes]

[Repeat for each theme]

## Pain Point Matrix
| Pain Point | Severity | Frequency | Evidence |
|------------|----------|-----------|----------|

## Design Opportunities
1. [Opportunity] — based on [evidence]
```

### ✅ Expected Output
- 4-6 themes organized by frequency
- Pain point matrix with severity ranking
- 3-5 actionable design opportunities

### 🔄 Iteration Tips
- ถ้า themes กว้างเกินไป → ขอให้ break down เป็น sub-themes
- ถ้าขาด quotes → ขอให้หา supporting evidence
- ถ้า opportunities vague → ขอให้ specific มากขึ้น
```
